Skills

Skills are what the agent CAN DO — discrete capabilities that extract data, validate criteria, or generate outputs.

ExtractEvidence

Reads clinical notes and extracts structured medical facts

Input → Output

PolicyRouter

Matches CPT code to the correct payer policy

Input → Output

GenerateNotice

Drafts the formal decision letter with citations

Input → Output
Skills are generated per-policy by the ClinicalNLP Engine based on each policy's criteria.
⚙️

Rules

Rules are governance guardrails — non-negotiable constraints that the agent MUST obey regardless of what skills produce.

RULE-01: PHI Redaction

Never expose patient names, SSN, or DOB in logs

Always Active

RULE-02: Clinical Conservatism

Never auto-deny — escalate uncertain cases to human

Always Active

RULE-04: Code Match

CPT/ICD-10 must align with the matched policy

Always Active
Rules are enforced at specific pipeline stages via Hooks. They cannot be overridden by skills.
🪝

Hooks

Hooks are lifecycle event triggers — they fire at specific pipeline stages and connect rules to the execution flow.

on_request_received → Triggers PHI redaction
on_guidelines_loaded → Triggers code validation
on_evidence_extracted → Validates extraction quality
on_criteria_evaluated → Applies conservatism rule
on_notice_generated → Enforces citations + plain language
Hooks are generated per-policy. Each policy type gets different hooks based on its specific validation needs.
🔌

MCP (Model Context Protocol)

MCP defines how the agent communicates with tools — a standardized contract for inputs, outputs, and resources.

Progressive Disclosure

The agent receives context incrementally — each stage only sees what it needs. This prevents context overload and enforces separation of concerns.

1 PHI rules only — no policies loaded
2 Coverage rules + member eligibility
3 Clinical guidelines for THIS CPT code
4 Rego evaluation rules (loaded NOW)
5 All rules disclosed — decision + notice
